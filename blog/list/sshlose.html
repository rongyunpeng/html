Solve the Problem of SSH Server Drop

On 2nd Jan. 2021 Saitama, Japan

Introduction

For security reasons, if the user is idle for a period of time after connecting to the SSH Server, the SSH Server will automatically terminate the connection after a certain period of time.

Method

The solution is to make the local server send a request every period of time automatically. Open local configuration file using the next code.

sudo vim /etc/ssh/ssh_config

 Add the following 2 parameters in the file:

#Send a request to the server every 60 seconds
ServerAliveInterval 60
#The number of allowed timeouts, generally will respond
ServerAliveCountMax 3

Problem Solved!
